# 🤖 課表查詢系統建置助手

當使用者啟動此技能，並提供其學校的課表檔案（PDF, Word, Excel 等），請 AI 助手依照以下標準作業流程（SOP）幫使用者建立或更新課表系統：

## 1. 讀取並分析課表資料
- 讀取使用者上傳的課表檔案。
- 解析該學校的課表結構，找出每堂課的「班級」、「星期幾」、「節次」、「科目」與「任課教師」。
- 若課表內有標示各班「導師」，請一併提取出所有班級的導師名單。

## 2. 轉換資料：建立教師課表 (CSV)
根據解析出的資料，透過 Python 或直接生成 `.csv` 檔（例如 `timetable_s1.csv`）。
**CSV 格式嚴格規範：**
1. 第一列 (Header) 開頭必須為：`teachername`。
2. 接著依序加入各節次欄位：`s11, c11, s12, c12, ..., s59, c59`。
   - `sXd`：代表星期 X 第 d 節的「科目」。(例如：s12 = 星期一第2節科目)
   - `cXd`：代表星期 X 第 d 節的「班級」。(例如：c12 = 星期一第2節班級)
   - （早自習可使用 `s10`, `c10`）
3. **資料轉換邏輯**：請將原始以「班級」為單位的課表，透過資料處理（樞紐分析）轉換為以「教師」為單位的橫式列表。每一列 (Row) 儲存一位教師整週的授課班級與科目。

## 3. 轉換資料：建立導師名單 (JSON)
建立對應的導師設定檔 `.json` 檔（例如 `homerooms_s1.json`）。
**JSON 格式嚴格規範：**
```json
{
  "101": "王大明",
  "102": "李小華"
}
```

## 4. 更新前端設定 (`config.js`)
編輯專案內的 `config.js`，依據使用者的資料進行客製化：
1. **學校資訊**：修改 `SCHOOL_NAME` 為使用者的學校名稱。
2. **學期設定**：在 `CONFIG.SEMESTERS` 區塊中，指向剛才生成好的 CSV 檔案（例如：`'114上學期': './timetable_s1.csv'`）。
3. **節次時間表**：檢查使用者提供的課表，如有明確的上下課時間，請同步更新 `PERIOD_TIMES` 陣列。

## 5. 完成通知
- 告知使用者系統資料已建置完畢。
- 提醒使用者可利用本地伺服器（如 `python -m http.server`）預覽，或協助將網頁推送部署至 GitHub Pages 或 Firebase Hosting。
